2017-5-6 django项目

搭建项目的命令

1.创建项目
django-admin startproject 项目名称
2.添加应用
python manage.py startapp bootest
3.在settings.py的配置文件里安装应用
在 INSTALLED_APPS(,,’booktest’)
4.同时需要将里面的语言切换为中文
即 LANGUAGE_CODE = ‘zh-Hans’
TIME_ZONE = ‘Asia/Shanghai’
5.启动简单的开发django的Web服务器
python mange.py runserver
6.指定数据库额，如果没定义数据库，就会默认使用项目目录下面的sqlite3的数据库
DATABASES = {
‘default’: {
‘ENGINE’: ‘django.db.backends.mysql’,
‘NAME’: ‘db_test’,
‘USER’: ‘root’,
‘PASSWORD’: ‘mysql’,
‘HOST’: ‘localhost’,
‘PORT’: ‘3306’,
}
}
6，在booktest文件夹下面的models文件里面创建类定义模型，也就是数据库里面的表
模型类是Python面向对象的程序，Python语言面向对象的程序一切皆对象，通过ORM的系统建立程序对象和数据库的映射关系，主要实现程序对象到关系数据库数据的对象映射。用python的程序语言直接操作数据库，避免了复杂的SQL语句，缺点是牺牲性能的，
7.创建好模型后，将文件迁移，将文件迁移为创建表的语句，
分为两步，一，生成迁移文件，python manage.py makemigrations
二， 执行迁移文件，python manage.py migrate

7,定义视图文件，在booktestm目录下views文件里定义，
.视图就是一个python函数，被定义在views.py的文件里.
.视图的第一个参数就是HttpReuest类型的对象的request,包含了所有的请求信息，
.视图必须返回HttpReuest对象，包含返回给响应者的所有信息。
调用模板分为三步骤都封装在render的函数里面的
1.找到模板
2.定义上下文
3.渲染模板
4.视图调用模板都要执行以上三部分，于是Django提供了一个函数render封装了以上代码
方法render包含3个参数
第一个参数为request对象
第二个参数为模板文件路径
第三个参数为字典，表示向模板中传递的上下文数据

.请求者在浏览器的地址栏输入url,请求到网站后，获取url的信息,然后配置好路径逐条匹配，
如果匹配成功就调用相应的视图，
一条URLconf包括url规则、视图两部分
url规则使用正则表达式定义
视图就是在views.py中定义的视图
需要两步完成URLconf配置
1.在应用中定义URLconf
2.包含到项目的URLconf中
